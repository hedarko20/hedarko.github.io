Explorations into Natural Language Processing

The Stanza NLP library is a Python natural language analysis library built by the Stanford NLP group and launched in 2020. One cannot speak about using Python for language processing without mentioning this up-and-coming contender in the fight for the most robust NLP library on the block. Using Universal Dependencies, the library’s consistent annotation is available in 60+ languages. In this article, I’d like to take a closer look at one of its main features, tokenization.

If you are unfamiliar with what tokenization is, feel free to catch up with my previous article: Explorations into Natural Language Processing: Introduction and the basics of Tokenization | by Zenzele Myricks | Sep, 2022 | Medium

Stanza NLP — Installation and Getting Set-up:

If you don’t already have Stanza on your machine, the following code will get you started.

To begin: install the Stanza NLP library. Stanza supports Python 3.6 or later. Installation via Python’s package manager, pip, is the most recommended route.

Using pip:

pip install stanza

If you already have the library installed, use the following code to ensure you have the latest version.

pip install stanza -U

For installation using Anaconda, use the following conda command:

conda install -c stanfordnlp stanza

According to the official documentation, Stanza via Anaconda does not currently support Python 3.8. For that version, use pip installation.

You also have the option of downloading via the GitHub repo for greater flexibility during app development on top of Stanza. To do this, you must first install Cuda. Second, you must install PyTorch. Once done, you may now run the following code:

git clone https://github.com/stanfordnlp/stanza.gitcd stanzapip install -e .

The next step is to initialize a pipeline. This will pre-load a series of processors, with each processor performing a specific NLP task. For general purposes, you can load the default English package and all processors for the language using this code.

import stanzastanza.download(‘en’)

Alternatively, we can specify the processor we need for the task at hand. Using the following code, we will load the default English package and specify the TokenizerProcessor as well as the POSProcessor to label our tokens with their proper entity information.

import stanzastanza.download(‘en’,processors=’tokenize,pos’)

Stanza NLP — Tokenize text data:

Sentence Segmentation and Tokenization

By default, the TokenizeProcessor simultaneously performs tokenization and sentence segmentation. We can use a ‘for loop’ and the built in Python function, enumerate(), to perform this operation on our document. Here is an example.

document = nlp(“BeBe Zahara Benet is from Cameroon. She is the first winner of RuPauls Drag Race.”)for i,sentence in enumerate(document.sentences):print(f’tokens in sentence#{i+1}:’)print(*[f’id: {token.id}\text: {token.text}’ for token in sentence.tokens], sep=’\n’)

Output:

The text has been segmented into its two sentences and each segment contains indexed tokens.

If you need just sentence segmentation, Stanza can also accommodate that. Here is an example.

document = nlp(“BeBe Zahara Benet is from Cameroon. She is the first winner of RuPauls Drag Race.”)print([sentence.text for sentence in document.sentences])

Output:

The return is a list of strings. The document has been split by its sentences only.

Tokenization without Sentence Segmentation

You may have documents that have been formatted prior to analysis and need to keep existing sentence segmentation. As long as within the raw text, the sentences are segmented by two continuous newlines, this can be done. Here is an example where we set the pipeline option “tokenize_no_ssplit” to True. This disables the default sentence segmentation and only runs tokenization.

nlp = stanza.Pipeline(lang=’en’,processors=’tokenize’, tokenize_no_ssplit=True)document = nlp(“BeBe Zahara Benet is from Cameroon.\n\nShe is the first winner of RuPauls Drag Race. She was also on All Stars 3”)for i, sentence in enumerate(document.sentences):print(f’ — Sentence {i+1} tokens — ‘)print(*[f’id: {token.id}\ttext: {token.text}’ for token in sentence.tokens], sep=’\n’)

Output:

This has returned two sentences split by the two newlines(\n\n) in the raw text. Both segments contain their respective indexed tokens.

Quick note about other features:

The Stanza NLP library can also be used with pre-tokenized text. Should you be using Stanza for other down streaming tasks, the library can still process your pre-tokenized data as newline (\n) separated sentences. The library also has a spaCy interface that can be used as the processor for tokenization. It has been implemented in use-cases with large-scale text data.

A Summary:

In this post we explored the TokenizeProcessor and how it can be used to return tokens, sentence segmentation and more. I can’t wait to use this in future pipelines. Thanks for joining me and stay tuned for more articles in my exploration into NLP.

项目目录如下

尝试

在 /retrieval/retrieval.py 中导入 /main.py 中的 TrainDataset 类。
一开始尝试 from main import TrainDataset 报错，找不到main模块。
会不会是应为当前目录是 main 的子目录，因此修改为 from ..main import TrainDataset 依然报错，不能相对索引。
那是不是项目没加到path里呢？于是修改为

import sys
sys.path.append('..')
print(sys.path)
from main import TrainDataset

依然报错，还是找不着。
把main.py 中的 TrainDataset分离到 /utils/dataset.py 还是报错。

解决

在打印的 sys.path 中确实存在 .. 这一项，可是为啥索引不到 main.py 或者 dataset.py 呢？
打印当前工作目录 print (os.path.abspath(os.curdir)) ，结果发现工作目录是 / ，但是运行 /retrieval/retrieval.py 时 Python 未将根目录加入 path 中。所以只需将 sys.path.append('..') 改为 sys.path.append('.') ，问题解决~
